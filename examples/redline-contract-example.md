# 示例 — Doc-Ops · 合同 Redline（v2 → v3）

## 场景

客户在投资谈判中收到投资人发回的 v3 合同。律师要快速看清相比 v2 改了哪些、哪些是重大变更。

## 用户输入

```
对方刚发回 v3 合同。原来发出去的是 v2。帮我对比一下，重点看变更是不是对己方不利。
（附 v2.docx 和 v3.docx）
```

## task-router 判断

| 维度 | 取值 | 分 |
| --- | --- | --- |
| 长度 | 50 字 + 2 个文档 | 1 |
| 领域数 | 1（合同） | 0 |
| 文档数 | 2 | 1 |
| 输出预估 | ~ 2000 字 | 1 |
| 文档关键词 | "对比"、"redline" | 触发 doc-ops 路由 |

→ **Medium → doc-ops 单 specialist**

## 编排

```
task-router
  → document-master
    → /redline-contract v2.docx v3.docx
       → 调 script.py 跑 diff-match-patch
       → LLM 二次按"金额/管辖/IP/保密/付款节点/通知地址/期限"分类风险
    → /audit-boilerplate v3.docx
       → 看 v3 是否漏掉标准条款
    → /check-consistency v3.docx
       → v3 内部金额 / 当事人名 / 日期是否前后一致
  → aggregator 整合为"v3 redline 报告"
  → checkers
  → 用户
```

## 期望输出（摘要）

```
# v3 vs v2 Redline 报告

## 摘要
- 🔴 高风险变更：3 处
- 🟡 中风险变更：5 处
- 🟢 利于己方：1 处
- 🔵 删除条款：2 处

## 逐条变更

### 第 3 条 付款方式 🔴
- v2：30/60/10（合同签订/交付/验收）
- v3：20/70/10
- 影响：首付下降，但验收前需投入 70% 资金，对己方现金流压力增大
- 建议：反提 30/60/10 或要求里程碑式付款

### 第 12 条 管辖 🔴
- v2：上海仲裁委
- v3：北京仲裁委
- 影响：北京仲裁对己方差旅成本与出庭便利度都不利
- 建议：争取保留上海

### 第 22 条 IP 归属 🟢
- v2：双方各自享有改进
- v3：明确创始团队保留商标
- 影响：利于己方

……

## 标准条款审计
- ✅ 管辖
- ✅ 仲裁
- ✅ 不可抗力
- ❌ 第三方受益人条款（v3 未约定）

## 一致性核查
- ✅ 当事人名称
- ⚠️ 第 5 条金额"100 万元"，附件 A 中为"￥1,200,000" → 建议确认

## 谈判要点（律师参考）
1. 死守第 12 条管辖 → 上海仲裁
2. 反建议第 3 条付款 → 25/65/10
3. 保留第 22 条 IP 归属 v3 表述

> 本报告由 contract-redliner + document-master 出具，律师签字盖章前请亲自复核。
```
