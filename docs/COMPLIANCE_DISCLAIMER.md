# 合规与免责 / Compliance & Disclaimer

## 一、本项目的法律属性

- **本项目是律师辅助工具**，不是法律咨询服务、不是律师事务所、不是法律科技产品的"输出方"
- 本项目的所有输出由**使用本项目的律师**对最终客户负责
- 项目作者及贡献者**不与最终客户构成任何形式的法律关系**

## 二、律师在使用本项目时的责任

1. **执业资格**：使用者应是中华人民共和国律师执业证书持有人
2. **复核义务**：任何对外文件（律师函 / 诉讼文书 / 合同 / 合规意见）必须由律师亲自审阅、签字、盖章
3. **利益冲突筛查**：客户数据进入本项目前，应通过律所利益冲突系统筛查
4. **保密义务**：客户数据进入本项目时，律师应确保符合《律师执业行为规范》与《保密义务》
5. **数据合规义务**：处理客户个人信息时，应符合 PIPL / DSL / CSL 要求；数据出境（包括传入境外 AI 服务）需评估合规路径

## 三、AI 输出的不确定性

- 大模型可能虚构条款号 / 案例文号 / 学说观点（俗称"幻觉"）
- 本项目通过 `citation-checker` agent 二次核查，但不保证 100% 准确
- **任何引用都必须由律师亲自核对官方原文**
- 法规更新频繁，本地 RAG 可能滞后

## 四、不适用的场景

本项目**不适合**用于：

- 紧急情况下的法律决策（如取保候审、强制措施期限内的决定）
- 涉及国家安全 / 国家秘密 / 商业秘密的特殊事项
- 直接对外发送 / 提交（必须经律师亲自审阅）
- 替代律师对客户的当面咨询

## 五、客户数据建议

- 客户真实数据进入本项目前**强烈建议脱敏 / 化名**
- 不将真实身份证号、银行账号、商业秘密细节灌入 Claude
- 涉及个人信息处理的应给客户履行 PIPL 告知 - 同意义务

## 六、地域限制

- 本项目聚焦**中国大陆法**
- 涉港 / 涉澳 / 涉台 / 涉外应触发 `cross-border-specialist`，律师应增加二次复核
- 不要把大陆法适用规则直接套用到港澳台或外国法

## 七、版本与时效

- 法规库依赖 `prepare-statutes.sh` 拉取，时效以下载日为准
- 项目 README / 各 agent / 各 skill 中提及的条款号、税率、社平工资倍数等以**律师执业时的最新法规**为准
- 任何"硬编码"在 skill 中的数字（如七级税率 / 经济补偿金计算公式）**仅为示意**，使用时律师必须用最新法规复核

## 八、本项目不分发的内容

- 不分发任何法规原文（避免版本不一致与版权风险）
- 不分发任何裁判文书原文
- 不分发任何客户数据 / 模板（templates 仅为框架，不含真实案例）

## 九、推荐工作流

```
客户接案
  → 利益冲突筛查（律所系统）
  → 客户数据脱敏
  → 进入本项目 (agentic-china-lawyer)
  → 律师亲自复核输出
  → 签字盖章对外
```

## 十、若发现问题

- 模型输出明显错误 / 虚构 / 违反执业规范 → 在 GitHub 提 Issue
- 法规更新未跟上 → 提 PR 更新 `prepare-statutes.sh` 或 `statutes/`
- 严重合规风险 → 邮件联系作者
